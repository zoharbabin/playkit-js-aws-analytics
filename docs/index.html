Architecture
How It Works
The plugin sits between the Kaltura Player and the Libra analytics framework, translating native player events into the exact CustomEvent format Libra expects.

Viewer

Clicks play, pauses, seeks, changes volume…

→

Kaltura Player

Fires native events: PLAYING, PAUSE, TIME_UPDATE…

→

AWS Analytics Plugin

Dedup, thresholds, seek filtering, payload formatting

→

Libra Framework

Listens for CustomEvents, handles network transport

3 CustomEvent Channels

Analytics

custom_aws_eb-analytics_notify-listener
Target: document
All 12 analytics events (Play, Pause, Watch, Complete, etc.) with video metadata.

State

custom_awstv_video-player-state
Target: document.body
Player state broadcasts: play, pause, ended. Fires every time (not deduped).

Logger

custom_aws_eb-logger_notify-listener
Target: document
Buffer duration metrics. Fires each time the player exits a buffering state.

UI Events API
HTML <!-- 1. Kaltura Player --> <script src="https://cdn.jsdelivr.net/npm/@playkit-js/kaltura-player-js@latest/dist/kaltura-ovp-player.js"></script> <!-- 2. Plugin (auto-registers as 'awsAnalytics') --> <script src="playkit-aws-analytics.js"></script> <script> const player = KalturaPlayer.setup({ targetId: 'player-container', provider: { partnerId: YOUR_PARTNER_ID }, plugins: { awsAnalytics: {} // That's it. videoId & videoTitle auto-resolve. } }); player.loadMedia({ entryId: 'YOUR_ENTRY_ID' }); </script>
JavaScript plugins: { awsAnalytics: { videoId: 'custom-id', // override auto-resolved entry ID videoTitle: 'Custom Title', // override auto-resolved entry name orgId: 'my_org', // default: 'awsm_tv' watchThresholdSeconds: 15, // default: 30 completeThresholdPercent: 0.90 // default: 0.95 } }
JavaScript // Get the plugin instance const plugin = player._pluginManager.get('awsAnalytics'); // Call from your custom UI controls plugin.sendUiEvent('SeekForward'); plugin.sendUiEvent('SeekBackward'); plugin.sendUiEvent('TranscriptToggle'); plugin.sendUiEvent('QualityChange');
Why no endpoint URLs or API keys?

This plugin is a CustomEvent bridge, not an HTTP transport. It dispatches browser events that the Libra framework (already on the page) listens for. Libra handles all network transport, authentication, batching, and retries.

Options
Configuration
All options are optional. videoId and videoTitle auto-resolve from the Kaltura Player's media info.
Option Type Default Description
videoId string auto Auto-resolved from player.sources.id. Set to override.
videoTitle string auto Auto-resolved from player.sources.metadata.name. Set to override.
orgId string 'awsm_tv' Organization ID in buffer-time logger events.
watchThresholdSeconds number 30 Seconds of watch time before Watch fires.
completeThresholdPercent number 0.95 Fraction of duration before Complete fires.
